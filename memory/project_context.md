Project: Credly

Purpose
Credly is a simple ledger and order management app used by a distributor to manage shops in a local area.

Primary User
The app is designed for a single user: the business owner (a distributor who visits shops and takes orders).

Business Workflow

1. The distributor visits local shops.
2. Shops place orders for products such as dry fruits, spices, and small packet items (e.g., ₹10 packets).
3. Orders are recorded in the app under each shop.
4. Items are prepared later in the warehouse (godown).
5. Orders are delivered to shops.
6. Shops may pay immediately or later on credit.
7. The app tracks how much money each shop owes.

Core Problems the App Solves

* Track how much each shop owes (credit ledger).
* Record new orders while visiting shops.
* Organize orders by shop.
* Quickly search shops and see their balance.
* Update balances when shops pay partially or fully.

Core Data Objects

* Shops
* Orders
* Products
* Transactions (credit / payment)
* Outstanding balance per shop

Key Features (Phase 1)

* Add and manage shops
* Search shops
* View how much each shop owes
* Add orders to a shop
* Add payments received
* Automatically update shop balance
* View order history for each shop

Future Features (Ideas)

* Import shop list from Khatabook PDF
* Shop location tracking
* Delivery route planner
* WhatsApp/Telegram bill sharing
* Shop order history page
* Product catalog management

Platforms

* Web app first
* Mobile-friendly UI so it works well on phone

Access Control
Single-user system.
Only the owner logs in and manages all data.

Important UX Requirement

The interface must be extremely simple because the user will operate it quickly while visiting shops.

Large buttons
Fast search
Minimal navigation
Mobile-friendly layout
